## 0. 什么是 Transformer

Transformer 是一种**完全基于注意力机制、不使用循环或卷积**的神经网络架构。由 Google 在 2017 年论文《Attention Is All You Need》中提出，最初用于机器翻译，后来演化为今天所有大语言模型（GPT、Claude、LLaMA 等）的底层结构。

### 一句话定义

> Transformer 是一个由编码器和解码器堆叠而成的序列到序列模型，每一层包含多头自注意力 + 前馈网络 + 残差连接 + 层归一化。输入一串词向量，经过层层变换，输出一串融合了上下文信息的词向量。

### 拆解关键词

**"完全基于注意力"** —— 整个模型的信息流动靠注意力来驱动。每个词直接"看到"所有其他词，不靠 RNN 的时间步传播，也不靠 CNN 的卷积核滑动。

**"不使用循环或卷积"** —— 彻底抛弃了 RNN 和 CNN。在当时这是最大胆的设计选择，因为所有人都认为处理序列数据必须用循环结构。

**"序列到序列（Seq2Seq）"** —— 输入一个序列（比如"我 爱 你"），输出一个序列（比如翻译后的英文，或者下一个词的预测）。

**"编码器 + 解码器"** —— 编码器把输入序列加工成深层表示，解码器根据编码器的输出生成目标序列。后来的 GPT 只保留了编码器部分。

**"多头自注意力 + FFN + 残差 + 归一化"** —— 这是每一层内部的四个标准组件。这本书目前已经讲完了"自注意力"，后面会依次讲剩下的三个。

### 想象一台机器

把 Transformer 想象成一台<span style="color: red; font-weight: bold;">由注意力驱动的信息精炼机</span>：

```
原始词向量 → [注意力混合上下文] → [FFN 做非线性变换] → 精炼后的词向量
                       ↑ 残差 + 归一化 ↑
             → [注意力再次混合] → [FFN 再次变换] → 更精炼的词向量
             → ... 重复几十层 ...
             → 最终输出：每个词都深度融合了全句上下文
```

每一层让每个词重新审视全句（注意力），然后重新理解自己看到的内容（FFN）。几十层下来，每个词从孤立的嵌入变成了高度语境化的表示。

### 和传统方法的区别

| 传统 RNN | Transformer |
|----------|-------------|
| 串行处理，词一个一个过 | 并行处理，所有词同时算 |
| 长句子容易忘前面 | 任意两个词直接建立联系（O(1) 信息路径） |
| 依赖时间步，GPU 利用率低 | 全是矩阵乘法，GPU 利用率极高 |
| 层数多了训练不稳定 | 残差连接 + 层归一化，可以堆几十上百层 |

### 本书的结构

这本书从自注意力的一个头开始，用"我 爱 你"三个词演示完整的计算过程，然后逐步加入多头、FFN、残差归一化、编码器解码器，最后拼成完整的 Transformer。

## 1. 问题是什么

让计算机理解"我 爱 你"这句话。

计算机只认识数字，不认识汉字。所以第一步，把每个词变成一个数字列表（向量）。比如：

```
"我" → [0.2, 0.8, -0.3, 0.5]
"爱" → [0.7, -0.1, 0.4, 0.2]
"你" → [-0.5, 0.3, 0.9, -0.6]
```

每个词用 4 个数字表示，这个"4"我们叫它 **d_model**。d_model 就是每个词向量的长度。

到这里，"我 爱 你"从三个汉字变成了三行数字。这是模型的输入。

---

## 2. 核心问题

"爱"这个词，孤立地看就是 [0.7, -0.1, 0.4, 0.2]。但在这句话里，"爱"是动词，它前面有主语"我"，后面有宾语"你"。

**模型需要让"爱"这个向量融合"我"和"你"的信息**，知道是谁爱谁。

Transformer 解决这个问题的方法叫**自注意力**，思路很简单：让每个词去看句子里的所有词，挑出跟自己相关的，吸收它们的信息。

---

## 3. 怎么判断"谁跟谁相关"

"爱"去检查句子里的每个词时，需要一个评判标准。"我"跟"爱"相关吗？非常相关。"你"跟"爱"相关吗？也很相关。

Transformer 的做法是：给每个词生成两个向量，一个用来"提问"，一个用来"被检查"。

- **Q（Query，查询向量）**：代表这个词"在找什么"
- **K（Key，键向量）**：代表这个词"是什么"

当"爱"想知道自己跟"我"多相关时，就用 Q("爱") 和 K("我") 做一个运算（点积），得到一个分数。分数越高，说明越相关。

---

## 4. Q 和 K 怎么来的

每个词进来的时候已经有一个向量了（长度 d_model）。拿它乘以一个矩阵，就得到 Q。乘另一个矩阵，就得到 K。

### W_Q 和 W_K 是什么

W_Q 和 W_K 是两个矩阵，里面的数字是**训练出来的**（一开始随机，训练中不断调整）。同一个输入乘不同的矩阵，得到不同的结果。

### 矩阵乘法的规则

- 输入向量：1 行、d_model 列
- W_Q 矩阵：d_model 行、d_k 列（d_k 是输出的维度，即 Q 的长度）
- 乘完的结果 Q：1 行、d_k 列

举例（d_model=4，d_k=3）：

```
输入 "我": [0.2, 0.8, -0.3, 0.5]  ← 1行4列

W_Q:                               ← 4行3列
[ 0.1,  0.5, -0.2 ]
[-0.3,  0.2,  0.4 ]
[ 0.6, -0.1,  0.3 ]
[ 0.2,  0.7, -0.4 ]

Q("我") = 输入 × W_Q = [-0.30, 0.64, -0.01]  ← 1行3列
```

### 计算过程（拆解每一维）

Q 的第 1 维 = 输入每个数字 × W_Q 第 1 列的对应数字，然后相加：

```
0.2 × 0.1   =  0.02
0.8 × (-0.3) = -0.24
-0.3 × 0.6  = -0.18
0.5 × 0.2   =  0.10
─────────────
加起来       = -0.30
```

Q 的第 2 维 = 输入每个数字 × W_Q 第 2 列的对应数字，然后相加：

```
0.2 × 0.5    =  0.10
0.8 × 0.2    =  0.16
-0.3 × (-0.1) =  0.03
0.5 × 0.7    =  0.35
─────────────
加起来        =  0.64
```

Q 的第 3 维 = 输入每个数字 × W_Q 第 3 列的对应数字，然后相加：

```
0.2 × (-0.2) = -0.04
0.8 × 0.4    =  0.32
-0.3 × 0.3   = -0.09
0.5 × (-0.4) = -0.20
─────────────
加起来       = -0.01
```

所以 Q("我") = [-0.30, 0.64, -0.01]。

同样的道理，用 W_K 算出 K("我")，用 W_V 算出 V("我")。对"爱"和"你"也各做一遍。

### W_Q 里的数字怎么来的

不是人设定的，是**训练出来的**。

- 训练刚开始：W_Q 里的数字随机填
- 模型读数据、做预测、看错了多少
- 错误反向传播，告诉每个数字该往哪个方向调
- 重复几万亿次，W_Q 收敛到一组好用的值

就像投篮——开始乱扔，每扔一次看偏了多少，调整动作，几万次后肌肉记住了。

## 5. 用 Q 和 K 算相关性分数

现在"爱"要检查它跟每个词的相关性。Q("爱") 分别去和每个词的 K 做**点积**。

点积就是对应位置相乘再相加：

```
score("爱" → "我") = Q("爱") · K("我")
score("爱" → "爱") = Q("爱") · K("爱")
score("爱" → "你") = Q("爱") · K("你")
```

得到三个分数，比如 [0.10, 0.02, 0.16]。分数越大，说明 Q 和 K 越匹配。

### 点积出来的分数有问题——softmax 登场

现在有了三个分数：[0.10, 0.02, 0.16]。能不能直接用它们当权重？

**不能。**三个问题：

1. 这三个数加起来不等于 1（0.10 + 0.02 + 0.16 = 0.28，不是 1）。我们需要权重是"每个词占百分之多少"。
2. 万一分数里出现负数怎么办？Q("爱") · K("我") 完全可能是负数，负数当权重没有意义。
3. 我们希望分数高的被明显放大，分数低的被压下去，模型敢于做选择，而不是每个人都差不多。

**softmax 就是解决这三个问题的。**

### softmax 怎么算

softmax 的公式很简单，两步：

```
第 1 步：每个数取 e 的幂（e 是自然常数，约等于 2.718）
第 2 步：用每个 e^x 除以所有 e^x 的总和
```

写成公式：`softmax(x_i) = e^x_i / (e^x_1 + e^x_2 + ... + e^x_n)`

### 用我们的例子手算一遍

分数是 [0.10, 0.02, 0.16]。

**第 1 步：每个数取 e 的幂**

```
e^0.10 = 2.718^0.10 ≈ 1.105
e^0.02 = 2.718^0.02 ≈ 1.020
e^0.16 = 2.718^0.16 ≈ 1.174
```

这一步做完，三个数都变成了正数（解决了问题 2，负数也能变成正的小数）。

**第 2 步：加起来，然后每个除以总和**

```
总和 = 1.105 + 1.020 + 1.174 = 3.299

1.105 / 3.299 ≈ 0.335   ← "爱"→"我" 的权重
1.020 / 3.299 ≈ 0.309   ← "爱"→"爱" 的权重
1.174 / 3.299 ≈ 0.356   ← "爱"→"你" 的权重
```

检查：0.335 + 0.309 + 0.356 = 1.000，加起来等于 1（解决了问题 1）。

### softmax 的直觉

注意看：原始分数差距并不大（0.10、0.02、0.16，最高和最低只差 0.14），但经过 softmax 之后，差距被放大了（0.335、0.309、0.356，最大值和最小值差了快 5 个百分点）。

这就像考试排名——原始分可能只差两三分，但 e^x 像一副"放大镜"，**把小的差距变成有意义的概率分布**，让模型能果断地更关注某个词、少关注另一个词（解决了问题 3）。

### 一句话总结 softmax

**把任意一堆数变成 (0, 1) 之间、加起来等于 1 的概率分布，同时放大差距。**

这就是"爱"分配给三个词的注意力权重。

## 6. 用权重去取信息——V 出场

### V 是什么

每个词还有一个 **V（Value，值向量）**，代表这个词"携带的实际内容"。V 也是用输入向量乘 W_V 得到的（和 Q、K 的生成方式一样，只是用的矩阵不同）。

假设通过 W_V 算出了三个词的 V（这里还是 d_k = 3，每个 V 是 3 个数字的向量）：

```
V("我") = [0.41, 0.53, 0.12]    ← 3 个数字的向量
V("爱") = [-0.20, 0.67, 0.44]   ← 3 个数字的向量
V("你") = [0.33, -0.15, 0.79]   ← 3 个数字的向量
```

### 加权求和——用矩阵乘法

现在"爱"拿到了注意力权重。这是一个行向量（1 行、3 列，3 列对应"我""爱""你"三个词）：

```
权重("爱") = [0.335, 0.309, 0.356]    ← 1×3 行向量
```

把三个词的 V 向量按行堆在一起，形成一个矩阵（3 行、3 列）：

```
        第1维  第2维  第3维
V("我") [0.41,  0.53,  0.12]     ← 第 1 行
V("爱") [-0.20, 0.67,  0.44]     ← 第 2 行
V("你") [0.33, -0.15,  0.79]     ← 第 3 行

这是一个 3×3 的矩阵
```

然后用矩阵乘法：权重行向量 × V 矩阵 = 输出行向量。

```
权重行向量 [0.335, 0.309, 0.356]
          ↑对应三个词："我"  "爱"  "你"

V 矩阵（3×3）：
        第1维  第2维  第3维
"我": [  0.41   0.53   0.12 ]    ← 行 = 哪个词，列 = 哪个维度
"爱": [ -0.20   0.67   0.44 ]
"你": [  0.33  -0.15   0.79 ]

矩阵乘法：
[0.335 0.309 0.356]  ×  V矩阵  =  [0.192 0.332 0.457]
    ↑ 1×3                 3×3              ↑ 1×3
 权重行向量                             输出行向量
```

矩阵乘法的规则：结果的每个位置 = 行向量逐位乘矩阵的对应列，再加起来。

**结果的第 1 维**（权重 × V 矩阵第 1 列）：

```
0.335 × 0.41  +  0.309 × (-0.20)  +  0.356 × 0.33
= 0.137  +  (-0.062)  +  0.117
= 0.192
```

**结果的第 2 维**（权重 × V 矩阵第 2 列）：

```
0.335 × 0.53  +  0.309 × 0.67  +  0.356 × (-0.15)
= 0.178  +  0.207  +  (-0.053)
= 0.332
```

**结果的第 3 维**（权重 × V 矩阵第 3 列）：

```
0.335 × 0.12  +  0.309 × 0.44  +  0.356 × 0.79
= 0.040  +  0.136  +  0.281
= 0.457
```

所以：

```
Output("爱") = [0.192, 0.332, 0.457]    ← 1×3 行向量
```

**输入是向量，输出还是向量**，维度不变。整个过程就是一个矩阵乘法。

### 这个结果到底是什么意思

算出来的 [0.192, 0.332, 0.457] 是"爱"吸收了上下文之后的**新表示**。拆开看每个维度里三个词的贡献，就能理解它在干什么。

看第 1 维（结果 = 0.192）：

```
来自"我": 0.335 × 0.41 = 0.137    ← 贡献最大
来自"爱": 0.309 × (-0.20) = -0.062
来自"你": 0.356 × 0.33 = 0.117
```

"我"的贡献最大（0.137），因为"我"的 V 在这一维上值很高（0.41），且权重也不低（0.335）。这说明第 1 维主要编码了"主语是谁"的信息。

看第 3 维（结果 = 0.457）：

```
来自"我": 0.335 × 0.12 = 0.040
来自"爱": 0.309 × 0.44 = 0.136
来自"你": 0.356 × 0.79 = 0.281    ← 贡献最大
```

"你"的贡献遥遥领先（0.281），因为"你"的 V 在这一维上高达 0.79，且权重 0.356 是最高的。这说明第 3 维主要编码了"宾语是谁"的信息。

**所以加权求和的实际意义是：**

- 权重（来自 Q·K → softmax）决定了**每个词的发言权**
- V 决定了每个词**发言的内容是什么**
- 加权求和就是让所有词按发言权大小，共同"投票"出一个新向量
- 这个新向量里，某个维度可能偏"我"（主语信息），另一个维度可能偏"你"（宾语信息）

"爱"不再只是"爱"了——它的新表示里嵌入了"主语是我、宾语是你"的上下文信息。

### 三个词一次性算完

上面只算了"爱"一个词。实际中，把三个词的权重摆成矩阵，V 也摆成矩阵，一次乘法就算出全部输出。

注意力权重矩阵。3 行 3 列，每行是一个词"关注各词"的权重分布，每列是"被某个词关注"的权重：

```
        关注→"我"  关注→"爱"  关注→"你"
"我"看:  [  ...      ...       ...  ]   ← "我"对三个词的关注权重
"爱"看:  [ 0.335    0.309     0.356 ]   ← "爱"的（我们算出来的）
"你"看:  [  ...      ...       ...  ]   ← "你"对三个词的关注权重
```

每个数字就是前面 Q·K → softmax 得到的结果。

V 矩阵（3×3，每行是一个词的 V 向量，每列是一个维度）：

```
       第1维  第2维  第3维
"我": [ 0.41   0.53   0.12 ]    ← V("我")
"爱": [-0.20   0.67   0.44 ]    ← V("爱")
"你": [ 0.33  -0.15   0.79 ]    ← V("你")
```

矩阵乘法（一次算出三个词的输出）：

```
  权重矩阵 (3×3)    ×    V矩阵 (3×3)    =    输出矩阵 (3×3)

"我"看: [w11 w12 w13]   [ 0.41  0.53  0.12]   [ out_11  out_12  out_13 ]
"爱"看: [w21 w22 w23] × [-0.20  0.67  0.44] = [ 0.192   0.332   0.457 ]
"你"看: [w31 w32 w33]   [ 0.33 -0.15  0.79]   [ out_31  out_32  out_33 ]
                                                       ↑ 3×3 矩阵
```

输出矩阵的每一行 = 权重矩阵对应那行 × V 矩阵。之前手算"爱"的那一行，就是拿权重矩阵的第 2 行 [w21 w22 w23] = [0.335, 0.309, 0.356] 去乘 V 矩阵，得到输出矩阵的第 2 行 [0.192, 0.332, 0.457]。

三个输出向量排在一起，就是这一层注意力的最终产出。

## 7. 这些操作到底在干什么——直觉理解

公式和计算会了，但为什么要这样设计？这一节不讲数学，只讲"意义"。

### 7.1 根本问题：一个词的意思取决于上下文

"爱"这个字，单独拿出来是一个模糊的概念。但在"我 爱 你"里，"爱"的意思是明确的——主语是"我"，宾语是"你"。

传统做法（比如 RNN）是让信息像接力一样从前往后传："我"的信息传给"爱"，"爱"再传给"你"。但这很慢，而且句子一长，前面的信息就丢了。

Transformer 的思路完全不同：**让"爱"同时直接去问"我"和"你"——不靠传递，靠自己主动去检索。** 这就是"注意力"这个名字的由来。

### 7.2 QKV 分别干什么——用图书馆类比

你去图书馆查资料，带着一个问题："我想了解唐朝的历史"。这是你的 **Q（查询）**。

每本书的封底有一段简介："本书涵盖唐太宗到安史之乱"。这是书的 **K（键/标签）**。

你把你的 Q 和每本书的 K 比对，决定每本书看多少——数学上就是 Q·K 求点积，然后 softmax 变成百分比。这是**注意力权重**。

最后你把各本书的**实际内容**（**V，值**）按比例融合，形成你的理解。

回到"我 爱 你"：

- "爱"的 Q 在问："谁在爱？被爱的又是谁？"
- "我"的 K 在说："我是主语" → 跟 Q 匹配度高 → 权重大
- "你"的 K 在说："我是宾语" → 也跟 Q 匹配 → 权重大
- "爱"自己的 K 在说："我是动词" → 匹配度一般 → 权重小

然后"爱"拿这些权重，把三个词的 V 按比例融合。V 里装的是词的实际内容——"我"的 V 携带第一人称信息，"你"的 V 携带第二人称信息，"爱"的 V 携带动作信息。

融合之后，"爱"的新表示不再是孤立的"爱"，而是"由'我'发起的、作用于'你'的爱"。

### 7.3 为什么不能 Q 和 K 用同一个向量

因为"要找什么"和"我是什么"是两回事。

同一个词"我"，既可能被别人查（此时贡献 K），也可能去查别人（此时贡献 Q）。如果 Q 和 K 用同一个向量，就强迫"我用来找别人的特征"和"我被别人找到的特征"一模一样。

现实中它们应该不同——"我"作为查找者时关心"谁是动词？"，作为被查者时标签是"我是名词、我是主语"。所以用 W_Q 和 W_K 两套不同的矩阵，让同一个词在主动和被动时展现不同的特征。

### 7.4 为什么还需要 V，而不能直接用权重去调原始向量

因为**关注谁**和**从那个词拿什么**也应该分开。

"爱"关注"我"很多（权重高）。但从"我"那里拿什么？拿主语属性？还是拿情感色彩？这取决于 V 编码了什么。

Q-K 决定了**看向谁**，V 决定了**拿到什么**。两个决定分开做，模型更灵活。

### 7.5 一句话总结：**权重是过程，不是产物。它决定了谁的信息有多少进入了输出。一旦进入，就已经是输出的一部分了，不需要单独标注来源。**

### 7.6 自注意力是定理还是工程方案

这是一个很好的元问题。答案是：**它是工程方案，不是数学定理。**

自注意力不是从某个公理推导出来的。没人能证明 QKV 必须拆成三个矩阵、softmax 一定是最优的归一化方法、d_k 必须等于 d_model/h。这些设计在 2017 年被 Google 的研究者提出，原因是：

1. 它解决了 RNN 的长距离遗忘和无法并行的问题
2. 它特别适合 GPU 的矩阵乘法硬件加速
3. 实验效果好——远远好于当时所有其他方案

先有效果，然后大家才慢慢分析"为什么有效"。很多深度学习的关键组件都是这样诞生的：凭直觉设计 → 跑实验效果好 → 别人复现也好 → 成为标准。理论解释往往是事后追加上去的。

所以不要把自注意力当成某种普适原理——它就是一个在当前硬件条件和数据规模下被证明极其有效的工程方案。这也是为什么"注意力"这个名字跟人脑的注意力其实没直接关系，只是一个好听的类比。叫它<span style="color: red; font-weight: bold;">自适应上下文混合器</span>可能更准确，只是没那么好记。

### 7.7 大模型到底怎么"用"这个向量

接续 7.5 的讨论——"爱"经过注意力处理后变成了一个向量，这个向量在不同句子里值不同。那么大模型内部是怎么"理解"这个功能的？

**第一：同样一个"爱"，在不同句子里输出的向量确实不同。**

| 句子 | 注意力偏重 | 输出向量 |
|------|-----------|---------|
| "我 爱 你" | "我"、"你"权重高 | 偏向主语+宾语信息 |
| "你 爱 我" | "你"、"我"权重高 | 同样是主语+宾语，但方向不同 |
| "爱 是 永恒" | "是"、"永恒"权重高 | 偏向抽象概念 |

同一个"爱"进来，因为注意力权重不同导致 V 的加权和不同，输出的向量不同。这是设计目标，不是意外。

**第二：大模型内部没有"理解"，它只有"拟合"。**

模型做的事本质上就一件——**预测下一个词**。训练时给它看"我 爱"，让它猜下一个词。猜对了"你"就奖励（W_Q、W_K、W_V 的数字往这个方向微调），猜错了就惩罚。

几十万亿次"猜→惩罚→微调"之后，W_Q 里的数字自动变成：当"爱"出现在第一人称主语附近时，Q("爱") 会跟宾语方向的 K 匹配出高分。没有谁告诉它语法规则——这些数字纯粹是数据喂出来的。

**第三：向量不需要被解释，只需要被用对。**

用调音师类比：你面前有几百个旋钮（参数），任务是听完前奏扭旋钮，让出来的声音跟原曲一样。几万遍后你成了大师，任何前奏进来你肌肉记忆般地扭出正确设置。但如果问你"第 47 号旋钮控制什么"，你答不上来。

大模型就是这个调音师。"爱"的注意力输出向量没有哪个维度单独代表"主语是我"。所有维度合在一起构成一个**分布式特征编码**——不需要人读懂，只需要后续网络层能利用它让正确的下一个词排到最高概率。

训练的目标就是：让语义结构相似的上下文产生相似的高维向量，让不同上下文产生的向量在高维空间里分开。这样最后一层 softmax 就能根据向量落在哪个区域，决定输出哪个词。

## 8. 整个过程总结（一个注意力头）

```
输入：三个词各自的向量（每个 d_model 维）

每个词 × W_Q → Q（用来提问）
每个词 × W_K → K（用来被检查）
每个词 × W_V → V（实际携带的信息）

对每个词：
  拿它的 Q 和所有词的 K 做点积 → 相关性分数
  分数做 softmax → 注意力权重（加起来等于 1）
  用权重加权求和所有词的 V → 这个词的输出

输出：三个词各自的新向量（每个还是 d_model 维）
```

整个过程里可以训练的东西就是三个矩阵里的数字：**W_Q、W_K、W_V**。

训练的目标是让这些数字变成：动词的 Q 能自动跟主语的 K 匹配出高分，让模型能正确理解"谁爱谁"。

## 9. 多头注意力（Multi-Head Attention）

前面 1~8 节讲的都是一个注意力头。现在加入多个头。

### 9.1 一个头的问题

一个头只能学**一种**关注模式。经过训练，这个头可能学会了"动词去找主语和宾语"。但语言里不止这一种关系。同一句话"我昨天在图书馆看了一本很有趣的书"里，同时存在：

- "看"的主语是"我"（主谓关系）
- "看"的宾语是"书"（动宾关系）
- "很"修饰"有趣"，"有趣"修饰"书"（修饰关系）
- "昨天"是时间，"图书馆"是地点（状语关系）

如果只有一个头，它只能平均地关注所有相关词，没法同时精细地捕捉每种关系。

### 9.2 多头的做法

多头 = 多个头**并行运行**，每个头有自己独立的 W_Q、W_K、W_V。

```
头 1: W_Q1, W_K1, W_V1 → 注意力计算 → 输出 1
头 2: W_Q2, W_K2, W_V2 → 注意力计算 → 输出 2
...
头 h: W_Qh, W_Kh, W_Vh → 注意力计算 → 输出 h

最终输出 = 输出 1 + 输出 2 + ... + 输出 h（拼起来）
```

训练后每个头会自然地"专精"于不同的模式——不是人指定的，是训练中自动分化的。

### 9.3 维度怎么处理——d_k = d_model / h

每个头用小维度。全模型维度是 d_model，头数是 h，则每个头内部的 Q、K、V 维度为 `d_k = d_model / h`。

比如 d_model = 4，h = 2，则 d_k = 2：

```
头 1: Q(4→2维), K(4→2维), V(4→2维) → 输出(2维)
头 2: Q(4→2维), K(4→2维), V(4→2维) → 输出(2维)

拼起来: 2 + 2 = 4 = d_model ✓
```

### 9.4 用"我 爱 你"举例（d_model=4, h=2, d_k=2）

**头 1**（用自己的 W_Q1、W_K1、W_V1）：

```
Q1("爱") = [ 0.15, -0.32]

K1("我") = [ 0.21,  0.08],  K1("爱") = [-0.11,  0.45],  K1("你") = [ 0.30, -0.19]
V1("我") = [ 0.52,  0.33],  V1("爱") = [ 0.18,  0.71],  V1("你") = [-0.44,  0.60]

Q·K → softmax → 权重 = [0.40, 0.25, 0.35]    ← 偏重"我"和"你"
加权 V → Output1("爱") = [0.10, 0.52]
```

**头 2**（用自己的 W_Q2、W_K2、W_V2）：

```
Q2("爱") = [ 0.48, -0.05]

K2("我") = [-0.30,  0.14],  K2("爱") = [ 0.25,  0.33],  K2("你") = [-0.08,  0.49]
V2("我") = [ 0.61, -0.20],  V2("爱") = [-0.35,  0.42],  V2("你") = [ 0.22,  0.78]

Q·K → softmax → 权重 = [0.25, 0.50, 0.25]    ← 偏重"爱"自己
加权 V → Output2("爱") = [-0.07, 0.36]
```

注意两个头关注了不同的东西：头 1 偏重"我"（0.40）和"你"（0.35），头 2 偏重自己"爱"（0.50）。

**拼起来**：

```
Output("爱") = [Output1 | Output2] = [0.10, 0.52, -0.07, 0.36]
                   ↑ 头1(2维)   ↑ 头2(2维)    ↑ 拼成 4 维 = d_model
```

**拼接之后：W_O 融合**

实际代码中还会再乘一个 W_O 矩阵（d_model × d_model）做一次线性变换。为什么要这一步？

拼接只是把两个头"物理上排在一起"，但信息仍然是隔离的：

```
拼完: [0.10, 0.52, -0.07, 0.36]
        └── 头1领地 ──┘  └── 头2领地 ──┘
```

前 2 维完全是头 1 的信息，后 2 维完全是头 2 的信息，互不交叉。如果不处理，下一层的头 1 只能看到前 2 维（只能看到主语信息），下一层的头 2 只能看到后 2 维（只能看到宾语信息）。下层头接收的信息被上层头的分组方式绑架了，每层都只能在同样的维度边界里工作。

W_O 就是打破这堵墙的：

```
拼完(1×4) × W_O(4×4) = 融合向量(1×4)
```

矩阵乘法之后，新向量的每个维度都是头 1 和头 2 信息的混合。下一层的每个头都可以从任意维度里自由提取自己需要的信息。

**头的独立只在"学习关注模式"这一步有意义。学完之后，每个词应该得到一个所有信息交织在一起的统一向量，而不是"头1区域 + 头2区域"的拼接体。** W_O 就是做这件事的。

W_O 里的数字也是训练出来的，和 W_Q、W_K、W_V 一样——开始随机，自动调整到最优。

### 9.5 头数 vs V 的维度——两个容易混淆的概念

**这两个概念完全不同，互不干扰。**

| 概念 | 问题 | 含义 |
|------|------|------|
| V 的维度（d_k） | "一个头里用几个数字描述一个词？" | 一个头内部的信息容量 |
| 头数（h） | "几个头在同时跑？" | 同时有多少套独立的注意力 |
| 两者关系 | d_model = h × d_k | 总维度 = 头数 × 每头维度 |

唯一的交集就是这个公式。除此之外，各管各的。

**本质理解：头数就是把 d_model 拆分成 h 组，每组 d_k 维各自独立算注意力。**

不是"8 个独立的人各拿一套 512 维数据"。而是把 512 维切成 8 段 64 维，8 段同时并行算。

### 9.6 头是怎么分组的

分组方式极其朴素——**按索引硬切，不学，固定死。**

```
d_model = 512，h = 8，d_k = 64

头 1 → 第 1~64 个维度
头 2 → 第 65~128 个维度
头 3 → 第 129~192 个维度
...
头 8 → 第 449~512 个维度
```

没有任何智能——就是 `Q[0:64]`、`Q[64:128]`、`Q[128:192]`... 纯按位置切。

那为什么训练后头 1 学会了主语关系、头 2 学会了宾语关系？因为每个头有自己独立的 W_Q、W_K、W_V。分组只是给人划了地盘，但每个人在自己的地盘上通过训练自动学会了最优的注意力行为。

### 9.7 为什么头会自动分工——梯度驱动的边际收益递减

"头 1 学会了主语关系、头 2 学会了宾语关系"——这些说法是人事后观察模型行为时**打上去的标签**，不是设计时就安排好的。

真实过程：

```
随机初始化 W_Q1, W_K1, W_V1（头 1）
随机初始化 W_Q2, W_K2, W_V2（头 2）
...
随机初始化 W_Q8, W_K8, W_V8（头 8）

  几十万亿次 "猜下一个词 → 错了惩罚 → 微调参数"

参数调着调着，自动出现了分工。
```

**为什么不会两个头都去学主语？**

这里涉及两个概念——损失和梯度。详细会在后面的"训练"章节展开，这里先用直觉理解：

- **损失**：模型预测和正确答案差了多少。差得多 → 损失大；差得少 → 损失小。
- **梯度**：对于某个参数（比如 W_Q2 里的一个数字），微调它一点点，损失会变大还是变小、变多少。

训练的过程就是不停计算每个参数的梯度，然后把参数往"让损失变小"的方向推。

现在假设训练进行到某一步，头 1 碰巧已经把主语关系学到了 95 分。头 2 此刻的参数也参与了主语关系的表达，但它做的只是在重复头 1 已经提供的信息——而头 1 已经把主语信息表达得很充分了。

这时对头 2 的参数来说：微调它们一点点，最终输出几乎不变 → 损失几乎不变 → **梯度接近零**。梯度不推这个方向。

反过来，宾语关系目前只学到了 30 分。头 2 的参数在这个方向上的微小调整就能显著降低损失 → **梯度的值很大**。梯度猛推这个方向。

**所以分工不是被"设计"的，是被梯度"筛选"出来的。**

梯度下降每次做的事就是：找出所有参数里"每调一点点、损失降得最快"的那些，优先推它们。头 1 已经搞定的方向，继续在那些方向上投入参数的边际收益接近零。头 2 的参数就只能往还有高回报的方向走。

### 9.8 类比：两个人在荒岛上觅食

头 1 无意中走到沙滩，发现了很多贝壳。头 2 也到沙滩，发现头 1 已经把贝壳捡得差不多了——剩下的很少，每次弯腰的收获很低。头 2 自然倾向于换个地方。走到树林，发现一颗椰子树。没人在竞争，捡一颗就能吃饱。头 2 就留在树林了。

没有谁指挥"你们要分工"。是资源分布的差异导致重复劳动的回报递减，推动每个人走向不同的方向。

### 9.9 最终真相：暴力训练 + 规模效应

很大一部分"智能"不是精心设计的算法，而是**简单结构 × 海量数据 × 海量算力**的涌现结果。Transformer 的精妙之处不在于"每部分都巧妙设计"，而在于它把结构拆得足够简单、足够并行，然后剩下的交给规模。

## 11. Q&A 记录

### Q: d_model 是什么？
A: d_model 就是每个词向量的长度（维度数）。比如上面例子里每个词是 4 个数字，d_model = 4。在实际大模型中，d_model 通常是 512、768、4096 等。它是整个 Transformer 统一的向量长度。

### Q: d_k 是什么？
A: d_k 是 Q 和 K 向量的长度。在单头注意力中 d_k 可以等于 d_model。在多头注意力中 d_k = d_model / 头数。

### Q: W_Q 和 Q 是什么关系？
A: W_Q 是矩阵（工具），Q 是它乘出来的结果。同一个输入向量，乘 W_Q 得到 Q，乘 W_K 得到 K，乘 W_V 得到 V。三个矩阵不同，三个结果也不同。

### Q: W_Q 的列数由什么决定？
A: 由你想要 Q 有几维来决定。想要 3 维的 Q，W_Q 就有 3 列。矩阵乘法规则：输入是 1×4，矩阵是 4×3，输出是 1×3。

### Q: W_Q 里的值是设定好的吗？
A: 不是。是一开始随机给，训练过程中靠反向传播一点点调整出来的。几万亿次"猜错→纠正→微调"之后，自动学到了好用的值。

## 12. 后续发展方向——工程优化视角

Transformer 的底层原理很简单，但暴力计算的成本极高。注意力复杂度 O(n²)，参数量随 d_model 和层数增长。业界大量工作在**不显著损失效果的前提下砍计算量和显存**。以下是主要优化方向：

### 12.1 推理加速：KV Cache

生成文本时，每输出一个新词就要重新算一遍全序列的注意力。但之前词算过的 K 和 V 是不变的——把它们缓存起来，每步只算新词的 Q、K、V，旧词的 K、V 直接复用。没有 KV Cache，推理慢到不可用。

### 12.2 显存与计算：FlashAttention

注意力计算最慢的不是矩阵乘法，是从显存读写中间结果。FlashAttention 把中间数据留在 GPU 芯片内部处理完，不写回显存。速度提升 2-4 倍，显存占用大幅下降。目前几乎所有大模型默认使用。

### 12.3 压缩 KV：GQA（分组查询注意力）

多个 Q 头共享同一套 K 和 V。比如 8 个 Q 头只配 2 组 KV 头，KV Cache 大小砍掉 75%（从 8→2），效果几乎不降。LLaMA 2、LLaMA 3 都用了这个技术。

### 12.4 降低精度：量化

参数精度从 FP16（16 位）压到 INT8（8 位）甚至 INT4（4 位）。显存砍掉一半到四分之三，推理速度翻倍，精度损失很小。这是让大模型能在手机上跑的关键技术。

### 12.5 稀疏激活：MoE（混合专家）

不是每个词都激活全部参数。把 FFN 层拆成 8 个"专家"，每个词只路由到其中 2 个。参数总量可以很大（几百亿），但每次前向传播只跑一小部分（几十亿）。GPT-4 和 Mixtral 都用了这个方案。

### 12.6 降低序列复杂度：稀疏注意力

不让每个词关注所有其他词。只看局部窗口内的词 + 几个全局关键位置。复杂度从 O(n²) 压到 O(n log n) 甚至 O(n)，处理超长文本（几万到几十万 token）时必备。

### 12.7 总结

```
注意力原理 = O(n²)，简单但重

优化方向:
  存储 ── KV Cache、GQA、量化
  计算 ── FlashAttention、稀疏注意力  
  参数 ── MoE、量化

所有这些都不改变注意力本身的计算逻辑，
只在工程层面决定"同一个架构是能用还是用不起"。
```
